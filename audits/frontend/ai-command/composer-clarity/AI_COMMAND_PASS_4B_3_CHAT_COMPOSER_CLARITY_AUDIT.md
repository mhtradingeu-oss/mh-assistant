# AI Command Pass 4B-3 - Chat Composer Clarity Audit

## Purpose

Audit the AI Command chat composer before improving its visual clarity.

## Current UX issue

The composer works technically, but it is not clear enough visually:

- The title/label is not strong enough.
- The input area can feel like a large generic text box.
- The user may not immediately understand what to type.
- The send action needs stronger visual hierarchy.
- The helper/status area should be easier to understand.

## Desired UX contract

The composer should clearly show:

- Who the user is chatting with.
- What the chat is for.
- A clear, readable writing area.
- A visible primary send action.
- Secondary actions that do not compete with send.
- A compact status/helper message.

## Safety constraints

- No backend changes.
- No route changes.
- No Task Center changes.
- No Workflows changes.
- No AI response logic changes.
- No durable task creation.
- No mutation behavior.

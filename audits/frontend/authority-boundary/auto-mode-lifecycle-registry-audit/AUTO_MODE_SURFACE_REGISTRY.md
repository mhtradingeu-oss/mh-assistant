# Auto Mode Surface Registry

| Surface | Route | Active or Dormant | Controller Owner | Subscription Owner | Start Trigger | Cleanup Status | Gate Behavior | Backend Mutation Risk | Lifecycle Risk | Recommendation |
|---|---|---|---|---|---|---|---|---|---|---|
| Publishing Auto Prepare surface | `publishing` | Active | `pages/publishing.js` via `ensurePublishingAutoModeBinding` -> `createAutoModeController` | `publishingAutoModeUnsubscribe` in `pages/publishing.js` | Explicit click on `publishingAutoPrepareBtn` | No explicit route-unmount unsubscribe call; guarded single subscribe while enabled | Uses engine waiting gate; approve/skip buttons call engine gate handlers | Medium (optional durable handoff writes from automation steps; publish endpoints separate) | Medium | Keep behavior; add lifecycle registry contract and teardown expectations before code changes |

## Registry Notes
- App-level route cleanup capability exists in `app.js` (`activeRouteCleanup`), but Publishing/Workflows route renders currently do not return cleanup callbacks for Auto Mode resources.
- Gate approve/skip in this registry means frontend runtime gating only; it is not backend governance approval.
- Any future surface activation must register create/start/stop/cleanup triggers explicitly before implementation.

# 05 — Visual Design Review

## Executive conclusion

The three executive rhythm pages are functionally strong, but none of them is fully launch-ready from a premium visual design perspective yet.

Governance is currently the closest to the desired final direction because it has the clearest structure, calmer rhythm, and better separation between authority, risk, and escalation. Home is useful but visually dense and still feels too much like an internal dashboard. AI Command has the strongest product concept, but the workspace is visually compressed and needs clearer separation between AI team, conversation, output, and execution areas.

The main visual issue is not missing decoration. The issue is hierarchy, density, spacing, and operating-surface consistency. The next design phase should simplify visible information first, then apply polish.

## Home visual review

### Current visual state
Home provides useful executive signals, but too many elements are visible at the same time. The page reads as a powerful internal dashboard rather than a calm executive operating surface.

### Main visual strengths
- Strong executive intent.
- Clear status/dashboard structure.
- Multiple useful operational signals are available.
- Next Best Action concept already exists.

### Main visual problems
- Too many cards and metrics compete for attention.
- The primary executive story is not visually dominant enough.
- The page feels dense and slightly zoomed.
- Secondary operational details are too visible.
- AI guidance does not yet feel like a trusted executive assistant layer.

### What should be simplified first
- Show fewer metrics by default.
- Promote one primary Next Best Action.
- Convert secondary status blocks into collapsible or lower-priority sections.
- Reduce competing cards in the first viewport.
- Rename some dashboard-like sections into executive operating language.

### Safest next visual improvement
Create a calmer top operating band:
- project readiness
- one risk/attention signal
- one next best action
- one AI guidance prompt

Then move detailed system health, recent activity, and secondary metrics lower or behind progressive disclosure.

## AI Command visual review

### Current visual state
AI Command has the strongest product idea because it can become the AI Team Command Center. However, the current layout feels compressed. The user can see the idea, but the hierarchy between specialist selection, conversation, output, and workflow is not calm enough.

### Main visual strengths
- Strong AI Team concept.
- Specialist workflow is valuable.
- Ask / Draft / Review / Route / Execute / Monitor rhythm is powerful.
- Good foundation for an AI operating room.

### Main visual problems
- Conversation, output, and workspace areas compete visually.
- Specialist/team selection is not dominant enough.
- Advanced tools and tabs create density.
- The page feels like a power-user workspace more than a guided executive AI room.
- It lacks the same executive status/header rhythm visible in Home and Governance.

### What should be simplified first
- Make AI Team / Specialist selection more visible and persistent.
- Separate the page into clear zones:
  1. AI Team context
  2. Conversation
  3. Output / Draft
  4. Workflow / Handoff
  5. Evidence / Memory
- Collapse advanced tools by default.
- Give the user one primary AI action at a time.

### Safest next visual improvement
Turn AI Command into an “AI Team Command Center” with a calm header, visible active specialist, one primary composer, and a separate output panel. Avoid adding more tools until the visual hierarchy is cleaner.

## Governance visual review

### Current visual state
Governance is the closest page to the target visual direction. It has a clearer command structure, stronger authority/risk separation, and better executive rhythm than Home and AI Command.

### Main visual strengths
- Clear governance mission.
- Stronger separation of authority, risk, blockers, and escalation.
- Cleaner than Home and AI Command.
- Good candidate for the canonical operating rhythm.
- Safer executive language.

### Main visual problems
- Some lower sections still feel like reports rather than decision surfaces.
- Evidence and supporting signals can become visually flat.
- AI role and limits should be more visually explicit.
- Needs stronger prioritization of “what requires attention now”.

### What should be simplified first
- Collapse lower audit/report sections by default.
- Promote authority owner, approval pressure, and escalation path.
- Make “AI prepare only / human approval required” visually clearer.
- Keep supporting evidence readable but secondary.

### Safest next visual improvement
Use Governance as the first visual operating-surface template:
- command header
- executive summary grid
- next best action
- safe execution path
- evidence/support area
- lower detailed reports collapsed

## Cross-page visual risks

1. **Density drift**
   Home and AI Command show too much at once. If this continues across pages, the system will feel complex even when it is powerful.

2. **Inconsistent header rhythm**
   Home and Governance have clearer operating headers. AI Command needs a matching orientation layer.

3. **Weak AI visual hierarchy**
   AI is present, but it does not always appear as a trusted operating layer. The AI role should be visible, bounded, and context-aware.

4. **Too many equal-weight cards**
   Many cards appear equally important. The final system needs stronger primary / secondary / supporting hierarchy.

5. **Report-like lower sections**
   Governance is good at the top, but lower sections can become flat reports. This risk exists across all pages.

6. **Action ambiguity**
   Buttons must visually distinguish:
   - navigation
   - AI guidance
   - draft/preparation
   - approval
   - execution
   - destructive actions

## Global visual rules before rollout

1. Every page needs a consistent operating header.
2. Every page should show one clear primary next action.
3. Every page should separate status, action, AI guidance, and evidence.
4. First viewport should be calm and decision-focused.
5. Secondary metrics should be collapsed, grouped, or moved lower.
6. AI areas must clearly state whether AI is preparing, suggesting, routing, or executing.
7. Use fewer badges and chips unless they directly support a decision.
8. Cards should not all have equal visual weight.
9. Lower report sections should be progressive disclosure by default.
10. Governance should be used as the temporary reference pattern, then improved and applied to Home and AI Command.

## Recommended next step

Do not start broad visual redesign yet.

First create the final executive rhythm decision summary that merges:
- Authority review
- Runtime safety review
- Executive cognition review
- Cross-page consistency review
- Visual design review

Then begin implementation in this order:
1. Governance as the first canonical operating-surface refinement.
2. Home simplification using the Governance rhythm.
3. AI Command transformation into AI Team Command Center.
4. Apply the stabilized rhythm to the remaining system pages.

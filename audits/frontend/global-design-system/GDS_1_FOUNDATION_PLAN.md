# STEP GDS-1 — Global Design System v1 Foundation

## Status
Implementation planned.

## Purpose
Create a clean, additive global design primitive layer for MH-OS as a premium AI Operations Platform.

## Direction
MH-OS should feel like:
- an AI Business Operating System
- an executive operating room
- a guided decision platform
- a calm, premium, intelligent workspace

## Scope
Additive CSS primitives only.

## Target File
`public/control-center/styles/mhos-executive-surface-primitives.css`

## Rules
- No JS changes.
- No backend changes.
- No route changes.
- No data changes.
- No deletion of old selectors.
- No `!important`.
- No page-specific hacks.
- No legacy selectors.
- Use existing tokens where possible.
- Preserve current page behavior.

## Intended Primitive Families
- OS page shell
- executive header / brief
- decision card
- attention cards
- operating path
- action panel
- AI panel
- evidence/details area
- soft motion utilities
- responsive composition helpers

## Adoption Rule
No page is redesigned in this step. These primitives will be adopted later page-by-page, starting with Home.

# 02 International Platform Vision

## Product Identity

MH-OS should feel like a premium AI Business Operating System: an intelligent control center where a business operator can understand state, choose the next best action, use AI specialists, trust source material, route work safely, and govern execution.

The frontend should communicate:

- AI Command as the operator's strategic interface.
- AI Team as specialist collaborators, not generic chat presets.
- Library as the source-of-truth authority.
- Tool Drawer as the structured capability and source handoff mechanism.
- Governance as the approval and policy authority.
- Operations centers as the work, queue, job, and signal backbone.
- Studios as controlled production workspaces.
- Integrations as the data and channel connection layer.
- Settings as the policy/defaults control plane.

## International Standard

The experience should work for global teams, agencies, operators, founders, and business functions across markets.

Desired qualities:

- Clear English operating language with no internal phase jargon on user-facing surfaces.
- Fast comprehension in under 10 seconds per page.
- One obvious primary next action per page.
- Consistent hierarchy: context, state, next action, workspace, rail, handoff.
- Compact professional density, not decorative dashboards.
- Strong safety clarity around what AI can prepare versus what humans/backend-approved actions execute.
- Reversible, inspectable handoffs between pages.

## Final Platform Feeling

The final platform should feel like:

- Home: executive command room.
- AI Command: AI team operations room.
- Library: source authority and evidence vault.
- Integrations: connection and data reliability center.
- Workflows: repeatable operating playbook center.
- Studios: production workbenches for campaign, content, media, publishing, ads, and research.
- Customer Center: customer intelligence and response safety center.
- Operations: execution control tower.
- Governance: authority, approval, and policy board.
- Settings: control plane.

## Standard Page Composition

Every non-Home route should converge toward:

1. Smart context header: page purpose, project context, readiness/status chips, one primary action.
2. Operating summary: 3-5 signals that matter for the page.
3. Main workspace: the actual work surface.
4. Right rail: source/provenance, AI guidance, next action, handoff, safety.
5. Clear destination map: where output goes next and what remains gated.

## Global Copy Direction

Use business operating language:

- "Prepare", "Review", "Route", "Approve", "Publish", "Monitor", "Sync", "Handoff", "Source", "Evidence", "Readiness", "Owner", "Risk", "Next action".

Avoid weak or generic language:

- "Section 1", "Section 4", "Dashboard", "Panel", "Suggested prompt" without outcome, "AI tools" without role/context.

## AI Power Visibility

AI power should be visible through useful operating outcomes, not visual decoration.

Each page should show at least one of:

- Which AI specialist can help.
- Which source/evidence AI will use.
- Which output AI can prepare.
- Which destination the output can be routed to.
- Which action remains human/backend-gated.

## Source-Of-Truth Visibility

Library should be treated as a platform-wide authority, not a file manager.

Every production page should make source context visible:

- Campaign source assumptions.
- Content source/provenance.
- Media reference assets.
- Publishing package source.
- Ads creative/source mapping.
- Research proof and evidence.
- Governance evidence requirements.

## Governance Visibility

Governance should be visible at the point of risk:

- Claims.
- Pricing/offers.
- Publishing.
- Customer communication.
- Ads.
- Privacy/tracking.
- Policy updates.

AI can prepare and summarize, but human/backend-governed authority must remain explicit.

## Safe Modernization Philosophy

Do not rewrite the frontend. Modernize through:

- Documentation-backed CSS authority.
- Shared primitives.
- Route metadata and copy improvements.
- Page-by-page visual convergence.
- Reversible class additions.
- Browser QA after each page.
- No backend/API/command behavior changes.

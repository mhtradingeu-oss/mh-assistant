# 07 Page By Page Upgrade Plan

## Principles

- Keep existing architecture.
- Patch one page or one global primitive at a time.
- Preserve IDs, route IDs, data attributes, handlers, API calls, and protected-write behavior.
- Do not touch `data/projects/**`.
- Do not delete features unless unused status is proven and documented.
- Validate after every patch with node checks and browser QA.

## Phase 2: Highest Visible Page Improvements

1. Content Studio: merge source/provenance, SEO, governance, and preview into cleaner main/rail hierarchy.

3. Research: standardize market evidence and destination handoff.
4. Insights: emphasize "what changed" and "what to do next".
5. Campaign Studio: minor density and action rail normalization.

## Phase 4: Preserve And Polish Strong Surfaces

1. Home: only refine topbar/sidebar alignment and density.
2. AI Command: only microcopy/source/tool-drawer polish.
3. Library: only header/source authority polish after selector audit.
4. Publishing: only shared shell alignment, no execution behavior changes.
5. Governance: only visual density and header alignment after policy mutation QA.
6. Operations routes: only copy/status/next-action clarity, no mutation changes.

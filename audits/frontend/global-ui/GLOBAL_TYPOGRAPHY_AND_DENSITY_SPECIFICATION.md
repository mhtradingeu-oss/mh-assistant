# GLOBAL TYPOGRAPHY & DENSITY SPECIFICATION

**MH-OS Executive AI Operating System — Visual Language**

---

## 1. Typography Philosophy

- **Executive Typography:** Minimal, bold, highly legible, never ornamental. Prioritizes clarity and instant scan.
- **AI Operating Typography:** Calm, intelligent, context-aware. Adapts to operational state, never shouts.
- **Operational Readability:** All text must be scannable at a glance, with clear hierarchy and no visual noise.
- **Compactness:** Surfaces and text are as tight as possible without sacrificing legibility. No wasted space.
- **Visual Calmness:** No aggressive scaling, no visual clutter, no unnecessary emphasis. Typography supports calm focus.
- **Avoid:** Enterprise dashboard fonts, oversized headings, cramped or generic SaaS text, noisy scaling, and visual clutter.

---

## 2. Global Typography Hierarchy

- **Page Titles:** 1.5–1.75rem, bold, tight line-height (1.2), single-line, ellipsis if overflow
- **Section Titles:** 1.25–1.5rem, semi-bold, line-height 1.2, single-line
- **Executive Metrics:** 2rem max, bold, monospace or tabular, no wrapping
- **Operational Metadata:** 0.85–1rem, semi-bold, muted color, single-line, ellipsis
- **Escalation Labels:** 1rem, bold, alert color only if active, single-line
- **Workflow Labels:** 1rem, regular, clear, single-line
- **AI Guidance Text:** 0.9rem, italic or lighter, muted color, single-line or two-line max
- **Passive Informational Text:** 0.85rem, muted, never bold, single-line
- **Button Typography:** 1rem, bold, all-caps or title case, tight padding
- **Sizing Rules:** Step down by 0.15–0.25rem per hierarchy level
- **Line-Height:** 1.2–1.35 for all text
- **Font-Weight:** Only page/section titles and metrics use bold; all else semi-bold or regular
- **Truncation:** All non-title text truncated with ellipsis if overflow
- **Wrapping:** Only page/section titles may wrap; all else single-line by default

---

## 3. Density System

- **Compact Density:** Used for workforce tiles, workflow steps, escalation, metadata. Minimal padding (4–8px), tight vertical rhythm.
- **Medium Density:** Used for section headers, AI guidance, passive info. Padding 8–12px.
- **Executive Density:** Used for page titles, executive metrics. Padding 12–16px, but never excessive.
- **Operational Density:** Used for action panels, orchestration surfaces. Padding 8–12px, tight grid.
- **Application:**
  - Compact: Default for all operational surfaces
  - Medium: Only for secondary/passive info
  - Executive: Only for top-level titles/metrics
- **Spacing Rhythm:** 4px base unit, multiples only
- **Surface Breathing:** No double padding, no wasted whitespace

---

## 4. Executive Scan Rhythm

- **Eye Movement Flow:** Top-down, left-aligned, primary to secondary, never zig-zag
- **Scan Pacing:** One dominant surface per scan zone, all else recedes
- **Visual Silence:** No visual noise, no unnecessary lines or dividers
- **Whitespace Doctrine:** Only enough whitespace for clarity, never for decoration
- **Orchestration Rhythm:** Workflow and escalation always visible, never overwhelming

---

## 5. Surface Compaction Rules

- **Compact Cards:** Max 2 lines of text, single-line preferred, tight padding
- **Compact Workforce Tiles:** Avatar + name + 1-line summary, all else collapsed
- **Compact Workflow Steps:** Icon + label, no extra description, overflow as pill
- **Escalation Compactness:** Icon + label, alert color only if active
- **Metadata Compression:** All metadata single-line, ellipsis if overflow

---

## 6. Visual Weight Hierarchy

- **Dominant Surfaces:** Page title, Health Strip, Next Best Action
- **Supporting Surfaces:** Workforce Room, Workflow Chain
- **Passive Surfaces:** Escalation Lane, Executive Snapshot, AI Guidance
- **Informational Surfaces:** Operational metadata, passive info
- **Border Usage:** Only for separation, 1px, muted color
- **Shadow Usage:** Only for dominant/active cards, subtle only
- **Glow Restraint:** No glow except for critical alerts
- **Contrast Hierarchy:** High contrast for dominant, medium for supporting, low for passive

---

## 7. Responsive Typography

- **Desktop:** Full hierarchy, all sizes as above
- **Tablet:** Step down all sizes by 0.1–0.15rem, truncate more aggressively
- **Mobile:** Step down all sizes by 0.2rem, only titles may wrap, all else single-line
- **Scaling Behavior:** No uncontrolled scaling, always stepwise
- **Truncation:** All non-title text truncated on tablet/mobile
- **Collapse Rules:** Secondary/passive surfaces collapse to icons or pills on mobile

---

## 8. AI Operating Identity

- **AI-Native Typography:** Dynamic, context-aware, calm, never ornamental, always operational
- **Legacy-Dashboard Typography:** Static, oversized, generic, noisy, or SaaS-like
- **World-Class Visual Rhythm:** Consistent hierarchy, tight compaction, clear scan, no visual noise

---

## 9. Motion Preparation Rules

- **Typography Motion Restraint:** No animated font size/weight changes
- **Hover Restraint:** Only subtle underline or color shift on actionable text
- **Transition Philosophy:** Only for actionable state changes, never for passive info
- **Operational Motion Pacing:** Motion only for workflow/escalation transitions, never for text

---

## 10. Runtime Safety

- **No Typography Override Wars:** All typography defined globally, never per-page
- **No Density Duplication:** Only one density system, no local overrides
- **No Uncontrolled Scaling:** All scaling stepwise, never fluid or percentage-based
- **No Page-Level Forks:** All pages use the same typography/density system

---

## 11. Strategic Recommendations

- **Immediate Priorities:**
  - Shrink all oversized headings
  - Compress all metadata and secondary text
  - Flatten all card and panel surfaces
  - Remove all legacy dashboard font patterns
- **What Should Become Smaller:**
  - Section titles, metadata, workflow labels
- **What Should Become Quieter:**
  - Passive info, AI guidance, escalation (unless active)
- **What Should Become Dominant:**
  - Page titles, Health Strip, Next Best Action
- **What Should Remain Passive:**
  - Executive Snapshot, AI Guidance, passive info

---

## 12. Success Criteria

- MH-OS feels premium, world-class, and calm
- Executives scan instantly, no visual noise
- AI feels operational, not ornamental
- Surfaces feel visually connected, not fragmented
- Orchestration feels effortless, never overwhelming
- Typography is consistent, clear, and modern


Operational breathing ensures:
- dense surfaces remain readable
- orchestration remains calm
- scan rhythm never feels compressed
- whitespace supports cognition
- compaction never becomes claustrophobic

Typography silence means:
- typography never competes for attention
- emphasis is rare and intentional

- recommendations remain explainable
- escalation states remain readable
- orchestration labels remain understandable
- operational reasoning remains visible
- typography supports confidence and clarity

Forbidden typography patterns:
- oversized SaaS headings
- decorative enterprise typography
- excessive bold text
- dense unreadable metadata
- multi-line operational noise
- animated typography
- visual font inconsistency

# AI Operating Home Blueprint

## Goal
Transform the Control Center from a page-based dashboard into a world-class AI Business Operating System experience without breaking the existing working pages.

## Core principle
Do not remove existing pages. Build a new AI Experience Layer that summarizes, guides, and routes users into the existing tools.

## Target user experience
The user should immediately understand:
- Current project status
- Highest-impact next action
- Active AI team members
- Pending approvals
- Active work pipelines
- Publishing readiness
- Ads and growth status
- Learning and optimization opportunities

## Main sections

### 1. Project Command Summary
Shows:
- Project readiness
- System health
- Launch blockers
- Pending approvals
- Active workflows
- Latest learning signal

### 2. Next Best Action
One clear primary recommendation:
- What to do next
- Why it matters
- Which specialist owns it
- Which page/tool will execute it

### 3. AI Team Panel
Visible specialists:
- Strategist
- Content Writer
- Media Director
- Publisher
- Ads Optimizer
- SEO Analyst
- Operations Manager
- Compliance Reviewer

Each specialist shows:
- Current focus
- Last recommendation
- Waiting action
- Quick command button

### 4. Work Journey Pipeline
Unified task journey:
Idea → Plan → Create → Review → Approve → Publish → Promote → Measure → Learn

Each stage links to existing pages:
- Plan: Campaign Studio / AI Command
- Create: Content Studio / Media Studio
- Review: Governance / Approvals
- Publish: Publishing
- Promote: Ads Manager
- Measure: Insights
- Learn: AI Command / Recommendations

### 5. Active Task Cards
Every task should show:
- Type: image, video, content, plan, ad, SEO, approval
- Owner specialist
- Status
- Next action
- Required approval
- Related asset or output
- Destination page

### 6. Intelligence & Learning
Shows:
- What the system learned
- What improved
- What failed
- What should be tested next
- Which data is missing

## Safe implementation strategy

### Phase 1
Create static blueprint and map current data sources.

### Phase 2
Add new Home sections using existing data only.

### Phase 3
Add AI Team cards with existing command routing.

### Phase 4
Add unified task pipeline using existing workflows, handoffs, approvals, media jobs, content items, publishing items.

### Phase 5
Polish design system:
- less clutter
- clearer spacing
- executive typography
- contextual action panels
- command-driven UX

## Risk control
- Do not rewrite backend.
- Do not remove existing pages.
- Do not replace page logic at once.
- Every new section should be read-only first.
- Add actions only after display is stable.

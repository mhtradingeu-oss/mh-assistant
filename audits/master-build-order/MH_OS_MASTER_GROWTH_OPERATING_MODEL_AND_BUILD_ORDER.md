# MH-OS Master Growth Operating Model and Build Order

## 0. Decision

MH-OS is not a collection of isolated pages or tools.

MH-OS is a unified AI Growth Operating System operated by a living specialist AI team.

The system must guide every project through the full growth loop:

Project Truth → Research → Insights / Reports → Strategy → Content → Media / Creative Production → Publishing / Ads Execution → Analytics / Learning → Optimization → Back to Strategy.

This document is the master operating model and build order for all future implementation phases.

---

## 1. Non-Negotiable Engineering Doctrine

All future work must follow this order:

1. Truth scan first.
2. Understand existing source, stash, audits, contracts, and runtime state.
3. Document the decision.
4. Apply the narrowest safe change.
5. Validate syntax, runtime closure, authority boundaries, and regression risks.
6. Commit exact files only.
7. Push only after verification.
8. Create closeout evidence.
9. Never rebuild an existing layer blindly.
10. Never use `git add .`.
11. Never pop stash blindly.
12. Never edit by blind sed/block replacement without context and validation.

## 5. Required Handoff Contracts

Each stage must hand off structured output to the next stage.

Required contracts:

- Project Truth → Research Brief
- Research → Insight Report
- Insight Report → Strategy Brief
- Strategy Brief → Content Brief
- Content Brief → Media Brief
- Media Brief → Publishing Package
- Publishing Package → Performance Report
- Performance Report → Optimization Brief
- Optimization Brief → Next Strategy Revision

No stage may proceed without the minimum required input contract.

## 6. Governance and Approval Rules

Human approval is required for:

- Publishing
- Sending customer messages
- Launching ads
- Changing prices
- Deleting or archiving important assets
- Changing source-of-truth data
- Using sensitive claims
- External provider execution with cost or public impact
- Any irreversible action

Human approval is not required for:

- Drafts
- Internal research summaries
- Strategy proposals
- Content variants
- Non-final creative concepts
- Internal recommendations

# Phase 9B Runtime Globals Classification Draft

## Categories

### A — Passive configuration/global registry
Exports static metadata or simple config only.

### B — Passive diagnostic/capture helper
Records/captures events but does not execute backend or mutate route authority.

### C — Neutralized legacy bridge
Preserves compatibility but returns safe neutralized responses.

### D — Active runtime authority
Actually controls routes, DOM, governance, AI, or backend execution.

### E — Suspicious active global risk
Requires exact load path and behavior proof before patch.

## Phase 9B Rule
No patch unless active runtime risk is proven.

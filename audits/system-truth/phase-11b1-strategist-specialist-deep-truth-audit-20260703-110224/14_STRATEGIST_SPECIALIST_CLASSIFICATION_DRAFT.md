# Phase 11B.1 Strategist Specialist Classification Draft

## Classification categories

### A — Strong / production-ready
The Strategist is safe, wired, data-grounded, useful, and has clear UX.

### B — Strong but needs UX polish
The core role and workflow exist, but the user experience can become more realistic and premium.

### C — Present but shallow
The role exists but outputs are template-heavy or not deeply grounded enough.

### D — Partially wired
Frontend exists, but backend, data, handoff, or owning workspace wiring is incomplete.

### E — Missing capability
A required Strategist capability is absent.

### F — Safety/compliance blocker
Improvement cannot proceed until authority or governance is clarified.

## Things to classify

- Profile/personality
- Chat behavior
- Voice readiness
- File/library/source input
- Strategy workflow
- Campaign brief workflow
- Launch plan workflow
- Funnel/offer positioning workflow
- Output preview quality
- Backend grounding
- Handoff to Writer / Campaign Studio / Workflows / Publisher / Operations
- UI/UX fit for a real strategy employee
- Tools and quick actions
- Safety boundaries

## Rule
No patch until exact gap and safe implementation plan are proven.

## Product Gaps

### Gap 1 — Premium Publisher Office UX is not final

Current surfaces exist, but the final Publisher should eventually feel like a publishing operations office:

- Publishing Readiness Board
- Channel Package Board
- Schedule Calendar
- Copy / Media / Compliance Dependency Board
- Approval Package Board
- Risk Checklist
- Channel Fit Review
- Final Package Preview
- Handoff Package
- Publishing Queue Monitor

Classification:
- Strong but needs UX polish.

### Gap 2 — Dependency checklist needs deeper product UX

The audit shows dependencies across content, media, approval, governance, and channel readiness.

But the final Publisher should make dependencies explicit and visual:
- content ready
- media ready
- claims checked
- approvals received
- channel selected
- schedule slot selected
- audience/CTA checked
- German copy checked
- asset format checked

Classification:
- Partially wired.

### Gap 3 — Channel readiness needs stronger structured view

Publisher can reason about channels and packages.

But a final channel readiness board should show each channel separately:
- Instagram
- TikTok
- Facebook
- Google
- Email
- Website
- Marketplace
- Ads handoff

Classification:
- Strong foundation, needs polish.

### Gap 4 — Approval package workflow needs stronger UX

Approval package logic exists as safe handoff/review.

But final UX should provide:
- required approver
- risk level
- claim notes
- missing proof
- final decision status
- governance link
- publish blocker reason

Classification:
- Partially wired.

### Gap 5 — Live publishing must remain gated

Backend publishing authority exists.

AI Command must never bypass:
- approval
- governance
- owner confirmation
- external channel confirmation
- schedule confirmation

Classification:
- Safety preserved.

### Gap 6 — Profile/Avatar/Voice customization is missing as a durable specialist system

Publisher has static role/profile definition, but no proven final user-configurable profile system for:
- display name
- avatar/profile photo
- male/female/neutral voice
- voice provider
- speaking style
- publishing style
- channel preferences

Classification:
- Missing capability.

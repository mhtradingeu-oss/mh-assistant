# Phase 11B.8 Compliance Reviewer Specialist Classification Draft

## Classification categories

### A - Strong / production-ready
The Compliance Reviewer is safe, wired, data-grounded, useful, and has clear UX.

### B - Strong but needs UX polish
The core role and workflow exist, but the user experience can become more realistic and premium.

### C - Present but shallow
The role exists but outputs are template-heavy or not deeply grounded enough.

### D - Partially wired
Frontend exists, but backend, data, handoff, approval, governance, policy, or owning workspace wiring is incomplete.

### E - Missing capability
A required Compliance Reviewer capability is absent.

### F - Safety/compliance blocker
Improvement cannot proceed until authority or governance is clarified.

## Things to classify

- Profile/personality
- Chat behavior
- File/library/source/handoff input
- Claim review workflow
- Approval risk workflow
- Publishing safety workflow
- Governance notes workflow
- Evidence/source requirement workflow

- Media asset compliance connection

- Backend governance / approval authority

- Whether AI can publish/remove flags or only recommend
- Overlap with Publisher / Operations / Ads / Writer
- Output preview quality
- UI/UX fit for a real compliance employee
- Tools and quick actions
- Safety boundaries

No patch until exact gap and safe implementation plan are proven.

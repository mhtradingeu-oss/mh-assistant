## Product Gaps

### Gap 1 — Premium Operations Office UX is not final

Current surfaces exist, but the final Operations Lead should eventually feel like a real execution office:

- Task Planning Board
- Workflow Draft Board
- Owner / Dependency Board
- Timeline Board
- Execution Checklist Board
- Queue Pressure Board
- Job Health Board
- Notification Triage Board
- Handoff Package Viewer
- Review Gate Board

Classification:
- Strong but needs UX polish.

### Gap 2 — Workflow execution authority needs dedicated audit

Backend workflow run and AI workflow run helpers exist.

But execution authority, confirmation lifecycle, audit logs, rollback behavior, and provider/worker triggering should be audited separately before expanding automation execution.

Classification:
- Partially wired or unproven for full live execution.

### Gap 3 — Task mutation workflow needs final confirmation model

Task creation/loading exists.

But final task mutation UX should define:
- who can create
- who can assign owner
- who can change due date
- who can close
- required confirmation
- audit trail
- rollback/reopen rules

Classification:
- Partially wired.

### Gap 4 — Queue / job / notification mutation boundaries need deeper audit

Task Center, Queue Center, Job Monitor, and Notifications exist.

But the final system should clearly separate:
- review-only state
- mark read
- retry
- route
- execute
- cancel
- escalate

Classification:
- Strong foundation, needs boundary audit.

### Gap 5 — Overlap with Customer Ops and Sales/CRM is controlled but should be audited globally

Operations Lead owns workflow/task/execution preparation.

Customer Operations Lead owns customer inbox, reply drafts, ticket drafts, SLA and escalation context.

Sales / CRM Lead owns lead qualification, outreach drafts, CRM summaries, and sales handoffs.

This is not harmful duplication, but it should be verified in the final full-team boundary audit.

Classification:
- Healthy overlap.

### Gap 6 — AI Command must not execute workflows or backend operations directly

Operations Lead must never bypass:
- user confirmation
- approval gates
- governance policy
- backend mutation guards
- owning workspace authority

Classification:
- Safety preserved.

### Gap 7 — Profile/Avatar/Voice customization is missing as a durable specialist system

Operations Lead has static role/profile definition, but no proven final user-configurable profile system for:
- display name
- avatar/profile photo
- male/female/neutral voice
- voice provider
- operations style
- default workflow style
- default priority rules
- escalation preferences

Classification:
- Missing capability.

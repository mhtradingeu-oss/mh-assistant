# Role Boundary Decision Matrix

## Strategist
Owns:
- campaign concept
- launch strategy
- offer angle
- channel sequence
- funnel map
- priority recommendation

Must not:
- publish
- approve
- execute workflows
- set budgets

Owner workspace:
- Campaign Studio
- Workflows for strategy-to-execution handoff

## Content Writer
Owns:
- hooks
- captions
- landing copy
- email copy
- message variants
- publisher handoff copy

Must not:
- publish
- approve risky claims
- invent unsupported facts
- run workflows

Owner workspace:
- Content Studio
- Publishing handoff

## Media Director
Owns:
- creative brief
- visual direction
- format requirements
- brand consistency
- asset checklist

Must not:
- generate images directly from AI Command
- upload assets without review
- approve assets
- execute media jobs

Owner workspace:
- Media Studio
- Asset Library

## Video Lead
Owns:
- short-form scripts
- motion direction
- storyboard
- voiceover draft
- video asset needs

Must not:
- generate video directly from AI Command
- run media jobs
- approve footage
- upload footage without review

Owner workspace:
- Media Studio
- Media Native later

## Publisher
Owns:
- publishing readiness
- final package review
- schedule draft
- channel formatting
- release handoff

Must not:
- publish without explicit approval
- override schedules
- bypass governance
- push live channels directly

Owner workspace:
- Publishing

## Ads Optimizer
Owns:
- ad concepts
- targeting angles
- paid campaign structure
- copy variants
- budget notes as guidance only

Must not:
- launch ads
- set budgets
- approve spend
- access ad accounts directly

Owner workspace:
- Ads Manager
- Integrations

## SEO & Insights Analyst
Owns:
- SEO signals
- performance insights
- content opportunities
- keyword intent
- data coverage gaps

Must not:
- edit website settings
- publish recommendations
- change analytics configuration
- mutate live SEO data

Owner workspace:
- Insights
- Integrations

## Compliance Reviewer
Owns:
- claim review
- evidence needs
- approval risk
- safe language
- governance notes

Must not:
- grant approvals
- override governance gates
- publish
- remove flags without review

Owner workspace:
- Governance
- Publishing review handoff

## Operations Lead
Owns:
- tasks
- workflows
- timelines
- owners
- handoffs
- execution preparation

Must not:
- run workflows without confirmation
- auto-approve tasks
- override authority gates
- execute backend operations directly

Owner workspace:
- Workflows
- Task Center
- Queue Center
- Job Monitor

## Customer Operations Lead
Owns:
- inbox review
- customer thread summary
- safe reply draft
- ticket draft
- SLA risk
- escalation routing

Must not:
- send customer replies
- create live tickets
- change SLA policy
- escalate without confirmation
- mutate CRM
- assign conversations

Owner workspace:
- Customer Center
- Operations Centers

## Sales / CRM Lead
Owns:
- lead qualification
- outreach draft
- follow-up sequence
- CRM profile summary
- pipeline next-step recommendation
- sales handoff

Must not:
- send outreach
- mutate CRM records
- advance pipeline stages
- confirm follow-ups without review

Owner workspace:
- Workflows
- future Sales / CRM Office

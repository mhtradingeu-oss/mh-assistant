# Phase 14A.1 — Protection Notes

## Must preserve

- Existing AI Command chat experience.
- Existing specialist definitions.
- Existing pages already built and improved.
- Existing Campaign Studio / Workflows / Publishing / Governance / Media / Content surfaces.
- Existing handoff system.
- Existing backend route authority.
- Existing public alias safety state from Phase 13.

## Must not do

- Do not create a new Strategist page if Campaign Studio or an existing workspace is already the correct authority.
- Do not duplicate ai-team-model definitions.
- Do not rename IDs blindly.
- Do not add a second chat system.
- Do not create new files unless the scan proves a missing layer.
- Do not patch backend/frontend in this phase.
- Do not change public alias routes.
- Do not add provider execution.
- Do not auto-run workflows.

## Design direction after scan

Strategist should become a premium AI Employee experience using existing AI Command + existing workspace authority, not a disconnected new page.

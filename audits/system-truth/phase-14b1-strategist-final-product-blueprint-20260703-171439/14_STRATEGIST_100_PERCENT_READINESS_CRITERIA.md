
Strategist 100% Readiness Criteria

Strategist may be considered complete for this phase only when:

AI Command
Strategist appears with clear role, summary, and placeholder.
Strategist chat behavior is guidance/preview/handoff oriented.
Strategist tools are available and clear.
Output preview routes correctly.
User can open the correct workspace.
No second chat exists.
Campaign Studio
Strategy War Room exists inside Campaign Studio.
User can see campaign intake.
User can see Strategy Canvas.
User can generate/preview Campaign Brief.
User can edit or request revision.
User can approve brief.
User can hand off to next specialist.
Existing Campaign Studio functions remain intact.
Handoff
Handoff includes campaign objective, audience, offer, channels, creative direction, constraints, and next destination.
Handoff routes reuse existing shared context.
Writer/Media/Video/Publisher/Operations can receive usable context.
Safety
No publish/send/workflow run/CRM mutation occurs directly.
Sensitive actions remain confirmation-gated.
Governance/compliance review is recommended where needed.
Local/provider
Local-first strategy is documented.
Paid provider use is optional and user-controlled.
No hidden provider execution.
Code health
No duplicate Strategist page.
No duplicate registry.
No ID rename.
Syntax checks pass.
No production route break.
No public alias safety regression.
UX quality
User understands what Strategist does.
User can start from chat or Campaign Studio.
User sees preview, edit, approve, and handoff clearly.
The workflow feels premium, simple, and not technical.

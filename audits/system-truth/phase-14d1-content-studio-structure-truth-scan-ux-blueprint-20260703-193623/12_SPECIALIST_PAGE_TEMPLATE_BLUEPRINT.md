# Specialist Workspace Page Template Blueprint

Every specialist-connected studio page should follow this structure:

1. Incoming Context
   What came from the previous page/specialist?

2. Main Owner Workspace
   What does this page's primary owner create or decide?

3. Output Builder
   What durable draft/output is created?

4. Preview
   How does the user see the output before handoff?

5. Readiness
   Is it draft, missing context, needs review, or ready?

6. Handoff
   Where should this go next?

7. Audit / Activity
   What source, role, status, and history should be visible?

## Role Principle
Specialists should be roles inside the workspace, not separate duplicate pages unless the workflow is truly separate.

## Crowding Principle
Use role tabs, compact cards, collapsible panels, or a clear sidebar. Do not place all specialist outputs in one long page.

## Authority Principle
The page owns the workspace, not the specialist. The specialist contributes to a role-specific panel.

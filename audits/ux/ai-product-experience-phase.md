# AI Product Experience Phase

## Goal
Transform the Control Center into a world-class AI Business Operating System frontend that is clear, guided, smart, and easy to use.

## Core Objective
Make every page answer:
1. What is this page for?
2. What is the current status?
3. What needs attention?
4. What can I do now?
5. What can the AI team do for me?
6. What is the next best step?

## Global UX Layers
1. Executive Header
2. Quick Create / New Workflow Launcher
3. Page Purpose Header
4. Status Summary
5. Main Workspace
6. Action Panel
7. Smart Team Panel
8. Bottom Docked AI Assistant
9. Language / Market Switch
10. Guided Step-by-Step Wizards

## Design System Targets
- Main content max width: 1280px
- Workspace max width: 1440px
- Reading content max width: 760px
- Modal width: 720px
- Wizard width: 960px
- AI Assistant width: 420px
- Spacing scale: 4, 8, 12, 16, 24, 32, 48, 64
- Breakpoints: 640, 768, 1024, 1280, 1536

## Smart Team Standard
Each page should include contextual AI support that can:
- Explain the page state
- Recommend the next action
- Prepare content
- Help edit
- Request approval
- Route to the right page
- Support publish/send/review workflows

## First Implementation Order
1. Global Header audit
2. Global + New Workflow Launcher
3. Bottom Docked AI Assistant
4. Smart Team Panel standard
5. Operations Center simplification
6. Library simplification
7. Publishing guided flow
8. Campaign Studio guided flow
9. Content / Media guided creation
10. Global language layer

## Safety Rules
- No backend rewrite
- No React migration
- No Auto Mode inside render
- No heavy intelligence loading on page open
- No page redesign without audit first
- Improve clarity before adding features

# Phase 1A-7 Route Access / Scope Matrix

“This Phase 1A-7 inventory freezes observed route, access, scope, principal, permission, authorization, security-enforcement, proof, isolation, transport-security, validation, denial, and audit vocabulary and records recommendations only. It does not approve route IDs, permission models, principal models, RBAC, scope inheritance, middleware changes, migrations, schemas, registries, or runtime changes.”

`requiredAccess` and `requiredScope` originate in `runtime/orchestrator-service/lib/security/route-permission-catalog.js:154-203`. They are classification metadata. The sensitive runtime middleware consults `requiredAccess` only after its own narrower regex match; no caller role/scope is resolved. `requiredScope` is diagnostic/log output, not an enforced principal grant.

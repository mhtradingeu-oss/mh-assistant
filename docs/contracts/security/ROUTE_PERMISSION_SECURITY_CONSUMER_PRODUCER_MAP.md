## Federated flow, not a universal pipeline

`request -> early path proof guard -> alias/transport middleware -> JSON parser -> shared-key guards -> narrow sensitive classifier/provider classifier -> rate limit -> project validation -> handler-local governance -> handler -> fragmented logs` is the nearest common ordering, but individual paths omit stages and late routes sit after the error handler. Authentication, subject resolution, role membership, effective permission calculation, CSRF, and universal audit correlation have no producer.

Authority-impact rule: frontend route and role producers remain projection-only; classifier scopes remain metadata; operational roles remain work ownership; only independently proved backend guards may deny execution.

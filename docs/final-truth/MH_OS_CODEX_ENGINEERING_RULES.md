# MH-OS Codex Engineering Rules

## Mandatory

- Scan before modifying.
- Preserve existing authority.
- Make minimal scoped patches.
- Add tests/evidence with changes.
- Validate before commit.

## Forbidden

- git add .
- broad refactors
- duplicate systems
- new registries without approval
- credential changes
- data migrations without plan
- frontend authority creation

## Workflow

Truth → Design → Patch → Validate → Commit

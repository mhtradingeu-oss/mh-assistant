# COMPONENT ARCHITECTURE

## Goal
Move toward reusable operational components.

---

# Future Component Types

## Executive Components
- metric cards
- operational status
- activity streams
- runtime indicators

## AI Components
- AI suggestion cards
- AI workflows
- command surfaces
- contextual prompts

## Workflow Components
- pipeline cards
- workflow stages
- publishing flows
- execution panels

## Navigation Components
- smart sidebar
- adaptive topbar
- command launcher
- workspace chips

---

# Rules

## Never
- duplicate UI ownership
- duplicate component logic
- mix runtime and rendering

## Always
- isolate components
- keep runtime external
- preserve accessibility
- support responsiveness
